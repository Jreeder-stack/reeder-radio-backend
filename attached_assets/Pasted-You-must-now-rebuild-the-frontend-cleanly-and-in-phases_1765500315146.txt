====================================================
PHASE 1 — FRONTEND ARCHITECTURE PLAN
====================================================
Design a modern, stable frontend architecture.

The design MUST include:

------------------------------------
CORE MODULES (NO UI INSIDE)
------------------------------------
src/audio/MicPTTManager.js
  - Single source of truth for microphone
  - Creates ONE mic track at startup
  - publish() and unpublish() functions
  - stop() ALWAYS stops the mediaStreamTrack
  - Guarantees no ghost-transmissions

src/audio/LiveKitManager.js
  - Handles joining multiple channels
  - Leaving channels
  - Managing subscriptions
  - Sending audio to correct room
  - Getting VU meter data

src/audio/ToneEngine.js
  - Emergency Tone A
  - Emergency Tone B
  - Emergency Tone C
  - Clear-air intermittent beep
  - Uses Web Audio API only

src/state/dispatchStore.js
  - Zustand store for:
      monitoredChannels
      primaryTxChannelId
      units
      emergencies
      patches
      channelLayout
  - Actions for updating each category

------------------------------------
UI LAYER
------------------------------------
src/pages/DispatchConsole.jsx
  - Defines layout:
      Left: units
      Center: drag-and-drop channel grid
      Right: emergencies, patches, events
      Bottom: PTT + tones

src/components/
  ChannelTile.jsx
  UnitPanel.jsx
  EmergencyPanel.jsx
  PatchPanel.jsx
  EventLog.jsx
  TopBar.jsx
  BottomBar.jsx

------------------------------------
STATE FLOW
------------------------------------
- UI → dispatchStore → LiveKitManager → MicPTTManager → ToneEngine
- No component should directly manipulate audio objects.
- All mic, tone, and room logic lives in the core modules.

------------------------------------
GRID SYSTEM
------------------------------------
- Use react-beautiful-dnd or dnd-kit
- Channel tiles must be draggable
- Layout must save to localStorage

------------------------------------
VALIDATION REQUIREMENTS
------------------------------------
- UI remains responsive even while monitoring multiple channels
- PTT cannot get stuck “on”
- Tracks must stop immediately on PTT releas
