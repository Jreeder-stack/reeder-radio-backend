Important Opus rules to follow
- The encoder must be fed exactly one valid Opus frame at a supported duration: 2.5, 5, 10, 20, 40, or 60 ms. For 48 kHz, valid frame sizes are 120, 240, 480, 960, 1920, 2880 samples per channel.
- The input PCM length must be frame_size * channels samples.
- The encoder state is stateful and must not be used for more than one stream at the same time, and must not be re-initialized for each frame.
- opus_encode() returns the actual encoded byte count; packets are variable length and transport must send only the returned byte count, not a fixed buffer length.
- Decoder output rate can differ from the encoder-side signal bandwidth/sample configuration; Opus supports interoperable decoding at supported output rates.

References already in repo/docs:
- opus_encode must get exactly one supported frame and frame_size is samples per channel.
