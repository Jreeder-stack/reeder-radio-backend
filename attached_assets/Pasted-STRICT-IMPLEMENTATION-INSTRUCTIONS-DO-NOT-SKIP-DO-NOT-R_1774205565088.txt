PHASE 3 — CREATE NEW RADIO CORE
Create a new native Android radio core that is independent of LiveKit.

Required Android-side components:
1. RadioAudioEngine
   Responsibilities:
   - coordinate capture/playback
   - start/stop transmit
   - start/stop receive
   - respond to floor-control events
   - keep state machine clean

2. AudioCapture
   Responsibilities:
   - capture microphone audio
   - feed PCM frames into encoder
   - support push-to-talk start/stop
   - avoid changing existing hardware trigger behavior

3. AudioPlayback
   Responsibilities:
   - receive decoded PCM
   - play incoming audio quickly
   - support low-latency playback
   - stop cleanly at TX end

4. OpusCodec or split into OpusEncoder / OpusDecoder
   Responsibilities:
   - encode mic PCM frames to Opus
   - decode received Opus frames to PCM
   - use voice-friendly settings
   - keep API clean and isolated

5. UdpAudioTransport (or transport abstraction with UDP implementation)
   Responsibilities:
   - send outbound encoded audio frames
   - receive inbound encoded audio frames
   - manage socket lifecycle
   - reconnect/rebind cleanly if needed

6. JitterBuffer
   Responsibilities:
   - smooth packet timing variation
   - prevent choppy receive audio
   - stay small enough for radio-style latency

7. FloorControlManager
   Responsibilities:
   - request transmit grant
   - handle granted/denied/busy
   - enforce single active talker per talkgroup unless emergency override rules apply

8. RadioStateManager
   Responsibilities:
   - track idle / receiving / requesting_tx / transmitting / reconnecting
   - publish observable state to existing UI/viewmodel/service

Server behavior:
- a radio joins a talkgroup via signaling
- when PTT is pressed, client requests floor
- server grants or denies based on channel state
- if granted, client sends encoded audio packets to relay
- relay forwards packets only to subscribers of that talkgroup
- on release, client stops sending and server marks channel idle
- emergency traffic may preempt normal traffic only if existing business rules allow it
